# 发布检查清单

这份清单定义了当前版本正式发布时能够承诺什么、哪些能力仍然保持 Beta，以及在升级、导入、导出、同步前如何保护用户数据。

## 当前 Stable 核心能力

以下能力属于当前正式稳定发布边界：

- 任务 CRUD
- 流记录 CRUD 与搜索
- 认证与多用户隔离
- 基础附件上传与读取
- JSON 导入导出
- 桌面端与 Android/Capacitor 的本地 SQLite
- API 服务器同步
- WebDAV 同步
- 升级迁移
- 基础设置

## 当前 Beta / 受限能力

以下能力暂不纳入稳定 SLA，不应成为唯一的数据恢复路径：

- AI 助手与 AI Agent 流程
- 服务端备份与恢复流程
- S3 同步与存储目标
- 窗口上下文
- 桌面小组件
- Think Session 与 Work Thread 模块
- 第三方插件生态

如果任一 Beta 功能失败，预期行为是核心 Todo 主流程和已有本地数据仍然可用。

## 发布门槛命令

每个候选发布版本在打 tag 前至少要通过以下命令：

```bash
pnpm lint
pnpm typecheck
pnpm test:all
pnpm --filter @my-little-todo/web build:vite
```

完整发布建议再补跑：

```bash
pnpm --filter @my-little-todo/mobile build
pnpm --filter @my-little-todo/admin build
cargo build -p mlt-server -p mlt-server-bin
```

## 升级前备份

任何升级、迁移或同步配置变更前，都建议先做备份。

1. 打开 `设置 -> 数据`。
2. 执行一次完整 JSON 导出。
3. 尽量把导出文件保存到应用工作目录之外的位置。
4. 如果附件很重要，请把导出文件与相关 blob 存储一起保留，或优先通过服务端导入路径恢复。
5. 确认导出文件可读后再执行升级。

## 导出与恢复

JSON 导出是当前唯一的完整恢复格式。

导出：

1. 打开 `设置 -> 数据`。
2. 触发 JSON 导出。
3. 确认文件中包含预期的任务、流记录、设置和 blob 元数据。

恢复：

1. 尽量先在干净环境或测试账户中恢复验证。
2. 打开 `设置 -> 数据`。
3. 导入之前导出的 JSON 备份。
4. 检查任务、流记录、设置和 blob 引用是否都存在。
5. 验证无误后，再用于正式恢复。

Markdown 导出仅用于阅读和转移，不是完整恢复格式。

## 发布验收

满足以下条件后，当前版本才算真正可发布：

1. 从旧版本升级后，任务、流记录、设置和 blob 元数据不丢失。
2. 升级过程中被中断后，再次启动能够恢复完成。
3. 导出 JSON 后，可在新环境完整恢复。
4. blob 引用与恢复后的元数据保持一致。
5. 本地 SQLite 与 API/WebDAV 同步在冲突时不会静默覆盖数据。
6. 关闭全部 Beta 能力后，核心 Todo 产品仍完整可用。
7. 任一 Beta 功能失败时，不影响核心数据与主界面。
8. Android/Capacitor 仍然走本地 SQLite 路径，不回退到 API store。

## 不阻塞首发的已知限制

- 仍有一个编辑器按需链路的大 chunk，但它不属于核心首屏入口。
- S3、服务端备份恢复、窗口上下文、桌面小组件和插件生态仍然属于 Beta 或受限能力。
- 桌面安装包升级流和 Android 真机升级流仍需在真实发布环境中做人工验收。
