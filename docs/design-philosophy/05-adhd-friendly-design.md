# ADHD 友好设计：心理学研究与功能体系

> 本文档记录了从心理学和神经科学角度对用户行为的深度分析，以及由此推导出的功能设计体系。
> 核心转变：从"管理任务"到"感知用户执行状态并自适应"。

---

## 一、用户状态分析

### 两种典型极端状态

通过对目标用户（即本产品的创造者/首要用户）的深度自我观察，识别出两种极端状态：

#### 状态 A：低能量态

| 表现 | 心理学对应 |
|------|-----------|
| 启动困难，什么都不想做 | **任务启动障碍（Task Initiation Deficit）**，超 70% 的 ADHD 成年人报告此问题 |
| 开始做一件事时推脱 | **执行功能障碍（Executive Dysfunction）**——意图与行动之间的桥梁断裂 |
| 只敢做之前能成功运行的事 | **风险规避 + 能力感需求**（SDT 中的 Competence），低能量时回避不确定性 |
| 拒绝新东西 | **多巴胺调节问题**——新事物的回报不确定，低能量时大脑拒绝冒险 |
| 脑海里想很多但不记录也不去做 | **Zeigarnik 效应的负面版本**——未完成想法占据工作记忆但启动成本太高无法外化，形成"认知拥堵" |

#### 状态 B：亢奋态

| 表现 | 心理学对应 |
|------|-----------|
| 开一堆 threads | **新奇追逐（Novelty Seeking）**——每个新项目启动阶段提供"新奇 + 挑战"的双重多巴胺 |
| 做完最有挑战的部分就不想做后续 | **Goal Gradient Effect 的逆向**——挑战消失后剩余工作变成低刺激任务，动力归零 |
| 不断开新的 | **蜜月效应循环（Honeymoon Effect）**——新系统/新项目的新奇感约 10-14 天后消退 |
| 所有 threads 都只做一半 | **ADHD 动力公式**：兴趣 × 紧迫 × 新奇 × 挑战，缺一则动力崩塌 |

### 这属于什么？

这两种状态的交替模式在心理学中有明确的分类归属：

- **ADHD 的拖延-超集中循环（Procrastination-Hyperfocus Cycle）**：注意力调节困难导致在"完全无法启动"和"过度集中于新奇事物"之间摆荡
- **执行功能障碍（Executive Dysfunction）**：前额叶皮层因多巴胺/去甲肾上腺素调节异常，无法为低刺激任务分配足够激活能量
- **超日节律（Ultradian Rhythms）**：人的精力以约 90-120 分钟为周期波动，峰值期可深度专注，低谷期应主动休息
- **情绪/能量调节困难（Emotional Dysregulation）**：不是情绪波动本身有问题，而是缺乏调节工具

**重要说明**：这些分析基于自我观察和文献对照。自我觉察是有价值的，但不能替代专业诊断。焦虑、抑郁、睡眠问题也可能产生类似表现。

## 二、心理学理论基础

### 与产品设计直接相关的理论

#### 1. Goal Gradient Effect（目标梯度效应）

**核心观点**：人越接近目标，付出的努力越大、动力越强。

**经典实验**：咖啡店忠诚卡实验——离免费咖啡越近的顾客，购买频率越高。

**产品启示**：
- 可视化任务进度，让用户看到"快完成了"
- 高进度任务应获得更高的推荐权重
- "收尾中"的任务应有醒目的视觉标识

#### 2. Endowed Progress Effect（赋予进度效应）

**核心观点**：给用户"已经开始了"的人工进度，能显著提升完成率。

**经典实验**：两组人分别拿到 8 格空白卡和 10 格已盖 2 章的卡（都需再盖 8 章）。后者完成率 34%，前者仅 19%，提升约 78%。

**产品启示**：
- 创建任务时自动标记为"已了解"阶段（不从零开始）
- 进度的视觉起点不是空的

#### 3. Zeigarnik Effect（蔡格尼克效应）

**核心观点**：未完成的任务持续占据工作记忆，产生认知张力。

**重要补充**：Masicampo & Baumeister (2011) 发现，仅仅"制定完成计划"就能释放这种张力，即使还没开始做。

**产品启示**：
- Brain Dump 模式的理论基础——把想法外化本身就有价值
- "记下来"能释放认知负担，即使不立刻执行
- Stream 的 spark 类型天然服务于此

#### 4. Implementation Intention（执行意图）

**提出者**：Peter Gollwitzer (1999)

**核心观点**：if-then 形式的计划（"如果到了 X 时间/情境，我就做 Y"）比单纯的目标意图效果显著更好。元分析效果量 d=0.61（中到大）。

**机制**：通过预先将行动控制委托给情境线索，赋予行动自动化特征，绕过执行功能障碍。

**产品启示**：
- `plannedAt` 字段的理论基础
- 锁定模式的"一旦选择就不回头"设计
- 计划板中把任务拖到具体时间段 = 形成 implementation intention

#### 5. Self-Determination Theory（自我决定理论）

**提出者**：Deci & Ryan

**三个核心需求**：
- **自主性（Autonomy）**：用户需要感到是自己在选择，不是被系统控制
- **能力感（Competence）**：用户需要感到自己是有效的、在变强
- **关联感（Relatedness）**：用户需要感到被理解、被支持

**产品启示**：
- 所有约束必须是用户主动邀请的（自主性）
- 进度追踪必须让人感到"我在变强"而非"我还差很远"（能力感）
- 应用的语气是温暖的教练，不是冷漠的监工（关联感）
- **关键警告**：行为改变应用容易优化"与工具的互动"而非"真正的行为改变"

#### 6. Commitment Device（承诺装置）

**来源**：行为经济学

**核心观点**：人可以通过预先限制未来的选择空间来对抗当下的冲动。经典案例：Odysseus 让船员绑住自己以抵抗塞壬之歌。

**产品启示**：
- 锁定专注模式的理论基础
- "选择被约束的自由"本身是自主性的最高形式（precommitment）
- 退出的"代价"应是摩擦而非惩罚

## 三、ADHD 友好 Todo 的设计原则

### 传统 Todo 的隐含假设 vs ADHD 现实

| 传统 Todo 假设 | ADHD 现实 |
|---|---|
| 用户有稳定的执行力 | 执行力随能量/兴趣剧烈波动 |
| 列出来就会去做 | 列出来反而增加焦虑和选择瘫痪 |
| 逾期标红能催人行动 | 逾期标红触发羞耻 → 回避 → 更不想打开 app |
| 连续打卡能建立习惯 | 断一天就全线崩溃，打卡变成焦虑源 |
| 按重要性排序有效 | ADHD 按"兴趣 × 紧迫 × 新奇 × 挑战"排序 |
| 更多功能 = 更好用 | 更多功能 = 更多决策 = 执行功能过载 |

### 从 ADHD 研究推导的设计铁律

1. **先展示成就，再展示待办**——"你已经完成了 3 件事"比"你还有 12 件事没做"有效 100 倍
2. **永远不惩罚**——没有红色逾期徽章、没有断裂的连续记录、没有让人觉得自己失败的 UI（与设计宪法第二条一致）
3. **降低到可笑的启动成本**——从打开 app 到开始做事，步骤越少越好（与设计宪法第三条一致）
4. **一次只给一件事**——不是列表，是"下一件事"（Now 视图已实现）
5. **微奖励、即时反馈**——每个小动作都有回应，不等到"全部完成"才庆祝（与设计宪法第七条一致）
6. **容忍不完美**——完成 5/7 天就是成功、做了一半也值得庆祝、搁置不是失败
7. **能量感知而非时间管理**——不问"你什么时候做"，而是"你现在能承受什么级别的任务"

### ADHD 用户的 Todo App 流失数据

- **60%** 的 ADHD 成年人在一周内放弃 todo app
- 典型曲线：第 10 天热情消退 → 第 14 天忘记打开 → 第 21 天彻底放弃
- 根本原因：新奇感消退后多巴胺回落，app 从"有趣的新工具"变成"又一个提醒我失败的东西"

### 对抗 21 天放弃的策略

1. **渐进式视觉变化**：随使用天数增加，视觉元素缓慢进化，维持"有什么不一样了"的微新奇感
2. **不惩罚缺席**：回来后的第一句话是"欢迎回来"而不是"你已经 5 天没打开了"
3. **随机鼓励和彩蛋**：完成时的庆祝语随机变化、偶尔的小惊喜，避免"看腻了"
4. **让应用成为收集品**：成就回顾、能量节律可视化，满足对"新发现"的持续兴趣

## 四、功能模块设计

### 总体架构：状态驱动的自适应系统

```
                    ┌───────────────────┐
                    │  能量状态声明      │
                    │  低 / 正常 / 高    │
                    └────────┬──────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
     ┌────────────┐  ┌────────────┐  ┌────────────┐
     │  温和模式   │  │  标准模式   │  │  守护模式   │
     │            │  │            │  │            │
     │ Brain Dump │  │ 计划板     │  │ WIP 限制   │
     │ 微步骤推荐 │  │ 进度追踪   │  │ 收尾提醒   │
     │ 单任务展示 │  │ 推荐+锁定  │  │ 切换检测   │
     │ 零压力     │  │            │  │            │
     └────────────┘  └────────────┘  └────────────┘
```

### 模块一：能量感知系统

**心理学基础**：Ultradian Rhythms、Self-Determination Theory（自主性需求）

**设计要点**：
- 用户手动声明，不做自动检测（保护自主性，避免侵入感）
- 三档：低电量 / 正常 / 高能量，默认"正常"
- 应用底部的能量指示器（像电池图标），随时可切换
- 切换后整个应用的行为模式改变

**各能量档位下的应用行为差异**：

| 维度 | 低电量 | 正常 | 高能量 |
|------|--------|------|--------|
| Now 推荐策略 | 只推小任务/熟悉任务/收尾任务 | 正常推荐 | 优先推"接近完成"的任务 |
| 信息密度 | 隐藏 DDL 倒计时/逾期标记 | 正常 | 显示 WIP 计数和切换警告 |
| 可用操作 | 隐藏"创建新任务"，突出 Brain Dump | 正常 | 创建前先检查 WIP |
| 专注模式 | 不推荐锁定（压力太大） | 可选锁定 | 建议锁定（防跳转） |
| 视觉调性 | 更柔和色调、更大留白 | 正常 | 正常，WIP 超限时变化 |

**数据层**：
- `behaviorStore` 新增 `energyLevel: 'low' | 'normal' | 'high'`
- 新增 `BehaviorEventType: 'energy_changed'`
- 推荐算法 `pickRecommendation` 接收能量状态参数

### 模块二：多阶段进度追踪

**心理学基础**：Goal Gradient Effect、Endowed Progress Effect、Zeigarnik Effect

**两种互补机制**：

**机制 A：阶段标记（Phase Progress）**
- 可选阶段：`了解 → 探索 → 动手 → 核心完成 → 收尾 → 完成`
- 不是所有任务都需要（小任务直接完成）
- 可跳跃但不可回退
- 视觉上用小型进度环，在任务卡片角落显示
- **Endowed Progress 技巧**：创建任务时自动标记为"了解"阶段

**机制 B：进展日志（Progress Log）**
- 用户随时追加简短的进展记录
- 与 Stream 的 `log` 类型打通——关联任务的 log 自动聚合到该任务的进展时间线
- 专注模式中的 notes 也计入
- 每条记录 = 微成就（轻量动画反馈）

**视觉表达**：
- 进度环用颜色渐变（灰→绿），不用数字百分比
- 强调"已完成的部分"而非"还剩多少"（正面框架效应）
- 进度超 70% 的任务有特殊标识

**数据层**：
- Task 新增 `phase?: TaskPhase`
- Task 新增 `progressLogs: ProgressLog[]`（含 `id`, `timestamp`, `content`, `source`）

### 模块三：锁定专注模式（Commitment Lock）

**心理学基础**：Commitment Device、Implementation Intention

**两层递进设计**：

| | 普通专注（现有） | 锁定专注（新增） |
|---|---|---|
| 进入方式 | 点击"开始" | 点击"开始" → 选择"锁定" |
| 退出成本 | 随时"搁置" | 写理由 + 30 秒冷却倒计时 |
| 可见范围 | 任务详情可查看 | 只有当前任务，无法切换视图 |
| 完成奖励 | 庆祝动画 | 更丰富庆祝 + "锁定完成"成就 |

**锁定中退出交互流**：
1. 显示"你已经坚持了 X 分钟"鼓励
2. 必须输入退出理由（至少 10 字）
3. 30 秒冷却等待（给理性大脑覆盖冲动的时间）
4. 冷却期间显示随机激励语
5. 冷却结束后可确认退出

**关键原则**：
- 锁定是**自愿选择**，不是系统强制（precommitment 原则）
- 退出的"代价"是摩擦而非惩罚
- 低能量状态下不推荐锁定
- 锁定中的 notes 自动成为 Progress Log

**数据层**：
- `FocusState` 新增 `locked: boolean`、`lockExitReason?: string`
- 新增 `BehaviorEventType: 'focus_locked_completed' | 'focus_locked_abandoned'`

### 模块四：计划板（Kanban）

**心理学基础**：Implementation Intention、Mental Contrasting

**列设计**（与进度阶段对齐）：
```
想法池 | 已计划 | 进行中(WIP限制) | 收尾中 | 已完成
```

- **想法池**：从 Brain Dump / Stream spark 来的未整理想法
- **已计划**：有明确定义但未开始的任务
- **进行中**：正在做的（有 WIP 限制，默认上限 3）
- **收尾中**：进度 ≥ 70% 或阶段 ≥ `core_done` 的任务
- **已完成**：最近完成的任务（展示"赢"的证据）

**与现有 Board 的关系**：
- 不替换现有 Board，作为 Board 的第三个子视图
- Board tabs：列表 | 日历 | **看板**
- 复用现有 `@dnd-kit` 基础设施

### 模块五：WIP 限制与收尾守护

**心理学基础**：Kanban WIP Limit、Goal Gradient Effect

**三层防护**：

1. **WIP 计数（被动信息）**：始终显示"进行中：N/限额"
2. **创建摩擦（主动提示）**：WIP ≥ 限额时，创建/开始新任务前温和提示并展示"差一点就完成"的任务列表
3. **切换频率检测（行为镜子）**：30 分钟内切换 5+ 次任务时温和提示，用户可选"这就是我想要的节奏"或"帮我专注一件事"

**收尾激励**：
- "收尾中"任务在推荐中有加权
- 完成收尾任务时强调"你完成了最后一步！"
- 周回顾中展示"收尾完成率"

### 模块六：Brain Dump 模式

**心理学基础**：Zeigarnik Effect（外化减压）、GTD Capture 原则

**交互设计**：
- 全屏/半屏纯输入界面，背景柔和
- 大输入框，placeholder："随便写点什么..."
- 回车 = 提交并清空（连续倾倒）
- 不需要任何分类/日期/优先级
- 所有内容进入 Stream（`entryType: 'spark'`）
- 退出时反馈："已捕获 N 条想法，以后再整理"

**进入方式**：
- 低能量模式下 Now 视图显示 Brain Dump 入口
- 任何模式下可通过快捷键触发
- 底部快速输入按钮长按 = Brain Dump

**与现有系统的关系**：
- 本质是 `QuickInputBar` 的"全屏批量版"
- 复用 `streamStore.create` 逻辑
- 新组件 `BrainDumpOverlay`

## 五、关于自由与约束

> 在这个设计里，用户是"自由"的吗？

**是的。** 系统的每一个约束都是用户主动邀请的：

- **能量状态**：用户自己声明，系统不自动判定
- **WIP 限制**：软性提示，可以说"是的，我就是要开新的"然后继续
- **锁定模式**：完全自愿进入，付出摩擦后仍然可以退出
- **Brain Dump**：零要求，想写就写
- **切换检测**：是"镜子"不是"警察"

唯一的"不自由"发生在用户**主动选择锁定**之后——但这是 **precommitment（预承诺）**：用"现在清醒的自己"去约束"未来冲动的自己"。选择被约束的自由，本身就是自主性的最高形式。

这与设计宪法第六条（"摩擦是设计工具"）和第十条（"推你一把但不是控制"）完全一致。

## 六、实现路线图

### 优先级排序原则

从"对用户自身的即时价值"和"实现复杂度"两个维度排序：

### 第一波：核心体验升级

1. **能量感知系统**——所有自适应行为的基础，实现成本低
2. **多阶段进度追踪**——解决最核心的"只有完成/未完成"问题，与 Stream log 天然契合
3. **Brain Dump**——在现有 QuickInputBar 基础上扩展，工程量小

### 第二波：执行力守护

4. **锁定专注模式**——在现有 FocusMode 上增强
5. **WIP 限制 + 切换检测**——在现有 behaviorStore 上扩展

### 第三波：全局规划

6. **计划板 Kanban 视图**——最大的新 UI 组件
7. **反 21 天放弃策略**——渐进视觉进化、随机鼓励、成就回顾

每一波做完后都是一个可用的、比上一版更好的产品。

## 七、待探索问题

以下问题在设计推进中需要回答：

1. 能量状态是否应该有时间维度？（如"这个低能量预计持续多久"）
2. 锁定模式的退出代价如何平衡？（太严格让人烦，太松没效果）
3. WIP 限制是全局的还是按角色的？
4. 进度阶段是全局统一模板还是每个任务自定义？
5. 能量状态历史的可视化对用户的价值有多大？
6. 如何检测"亢奋态"下的切换行为而不误判正常的多任务处理？

- Todo app 一周内放弃率（ADHD 用户）：60%
- 新奇效应消退典型周期：10-21 天
- 能量/注意力的超日节律周期：90-120 分钟

### ADHD 友好 Todo 应用参考

- **Norakind**：Daily Focus（每日最多 3 件）、"Why does this matter TODAY?" 框架
- **Tasklr**：5-15 分钟任务块、温和提醒、游戏化成就
- **ConvoAct**：Spoon Budget System（能量追踪）、情绪日志、Body Doubling
- **FocusLock / CommitNow**：锁定模式、不可暂停、视觉紧迫感设计
