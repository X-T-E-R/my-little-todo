# 桌面小部件（Desktop Widget）设计规范

> 本文档是桌面小部件的设计宪法。所有迭代都必须遵守这里的原则。
> 与主应用设计宪法（`00-design-constitution.md`）完全对齐。
> 其他插件/模块的设计原则索引见 [plugins/README.md](./README.md)。

---

## 零、小部件与主应用的关系

小部件**不是主应用的缩略版**，而是一个**独立的微型交互面**。

| 维度 | 主应用 | 小部件 |
|------|--------|--------|
| 使命 | 完整的执行系统：记录、计划、执行、回顾 | **瞥一眼就知道该做什么**，然后回去干活 |
| 交互时长 | 数秒到数十分钟 | **< 3 秒**（扫视）或 **< 5 秒**（操作） |
| 认知负担 | 允许适度复杂（有导航、面板、详情） | **零决策**——看到的就是答案 |
| 视觉存在 | 独立窗口，用户主动切换 | 桌面常驻，融入背景又不消失 |

**核心隐喻**：小部件是贴在显示器边框上的**便利贴**——你不会主动去「使用」它，但你的余光随时能扫到它。

## 一、设计铁律

### 铁律 1：信息密度 = 一屏到底，不滚动

小部件内容必须在单屏内完整展示。需要滚动 = 设计失败。
- 任务列表最多 **5 条**（不是 7 条、不是 10 条）
- 不显示已完成、不显示归档
- 如果当前角色有 20 个活跃任务，小部件只展示排序最高的 5 个

### 铁律 2：不制造焦虑

与设计宪法第二条一致：
- **不显示未完成总数**（不显示「还有 12 件事」）
- **不显示逾期标记**（不用红色、不用感叹号）
- **不显示倒计时**（数字天数会产生压力）
- 唯一的时间暗示：DDL 最近的任务排在最前面（位置即信息）

### 铁律 3：视觉融入桌面，不抢夺注意力

小部件必须在用户「不看它」时近乎隐形，在用户「看向它」时清晰可读。
- 默认状态使用**极低对比度背景**，配合高模糊
- 文字使用半透明色而非纯黑/纯白
- **不使用彩色图标、不使用分割线、不使用卡片阴影**
- 鼠标悬停或获得焦点时提升对比度（从 whisper → speak）

### 铁律 4：拖拽区 ≥ 50% 面积

桌面用户最高频的操作是移动小部件位置。
- 标题栏和空白区域全部是拖拽区
- 只有按钮和任务文字是非拖拽区
- 拖拽不应有惯性/弹性动画（桌面工具需要精确定位）

### 铁律 5：操作不超过两个

小部件上能做的事：
1. **切换显示模式**（Overlay / Pin）
2. **点击任务 → 打开主窗口对应详情**

不放在小部件上的操作：
- 创建任务（去主窗口或用全局快捷键）
- 完成任务（需要确认，去主窗口）
- 编辑任何内容

## 二、视觉结构

### 纵向三段式布局

```
┌─────────────────────────────┐
│ ⬡ 角色名 · 状态         ⠇  │ ← 头部：当前角色 + 窗口上下文状态
├─────────────────────────────┤
│                             │
│  ① 任务标题（截止最近）       │ ← 身体：5 条任务，紧凑排列
│  ② 任务标题                 │    每条一行，无边框无卡片
│  ③ 任务标题                 │    DDL 最近的排前面
│  ④ 任务标题                 │
│  ⑤ 任务标题                 │
│                             │
├─────────────────────────────┤
│  下午好 · 精力正常           │ ← 脚部：问候 + 能量状态（可选）
└─────────────────────────────┘
```

### 尺寸约束

| 参数 | 值 | 原因 |
|------|------|------|
| 默认宽度 | 280px | 刚好容纳一行中文约 14 字 |
| 最小宽度 | 240px | 再窄文字会过度截断 |
| 最大宽度 | 360px | 再宽就不是小部件了 |
| 默认高度 | 自适应内容 | 不固定高度；内容少时窗口小 |
| 最大高度 | 屏幕高度 50% | 避免遮挡过多 |
| 圆角 | 16px | 与主应用一致 |
| 外边距 | 8px（透明窗口内） | 保证投影不被裁切 |

### 颜色与透明度

**核心原则**：小部件是「呼吸在桌面上的薄纱」。

| 状态 | 背景 | 文字 | 边框 |
|------|------|------|------|
| 默认（静止） | `surface` @ 75% opacity + 24px blur | text @ 85% | border @ 40% |
| 悬停 | `surface` @ 90% opacity + 16px blur | text @ 100% | border @ 60% |
| 暗色主题 | 同上但使用暗色 token | — | — |

**渐变微光**：头部区域使用一条 1px 高的渐变线（accent 色 → 透明），替代传统分割线。

## 三、内容规则

### 头部

- 左侧：角色颜色点（6px 圆形）+ 角色名（12px 半粗）
- 右侧：模式切换按钮（图标 14px，灰度，悬停才显色）
- 如果窗口上下文匹配了特定规则：角色名下方显示一行 10px 灰色文字（匹配的备注摘要）

### 任务列表

- 每行：`状态色块（3px 宽竖条）· 任务标题（12px）`
- 状态色块含义：
  - `accent` = today
  - `text-tertiary` = active（正常）
  - `warning` = DDL 在 3 天内（不用红色，用暖色）
- 标题单行截断（`text-overflow: ellipsis`）
- 悬停时标题行背景微微提亮
- 点击某行 → 打开主窗口并导航到该任务详情

### 脚部

- 左侧：根据时段动态问候（早上好 / 下午好 / 晚上好）
- 右侧：能量状态文字（低 / 正常 / 高），与主应用联动

### 空状态

当无匹配任务时：
- 不说「没有任务」（暗示你应该有）
- 显示：「此刻无事 · 享受安宁」（与设计宪法第十条一致）

## 四、动效规范

| 场景 | 动效 | 参数 |
|------|------|------|
| 窗口出现 | fade in + 轻微上移 | opacity 0→1, y 8→0, 200ms ease-out |
| 任务列表切换（角色变化） | 列表整体 crossfade | 150ms |
| 鼠标进入 | 背景不透明度提升 | 120ms linear |
| 鼠标离开 | 背景不透明度恢复 | 300ms ease-out（慢出快入） |
| 点击任务行 | 行短暂高亮 → 跳转 | 80ms pulse |

**禁止的动效**：
- 弹跳、回弹、spring（与「精确定位」冲突）
- 持续循环动画（分散注意力）
- 3D 透视变换（性能开销，与透明窗口兼容性差）

## 五、技术约束备忘

- 透明窗口 + backdrop-blur 在 Windows 上依赖 `Mica` / `Acrylic` 支持；Tauri `transparent: true` + CSS `backdrop-filter: blur()` 即可
- `data-tauri-drag-region` 必须覆盖所有非按钮/非文本区域
- 小部件加载自己的 DataStore（与主窗口独立的 webview），需要在 `WidgetShell` 中完整初始化存储
- 窗口标题不可见（`decorations: false`），但 `title` 字段需要填（辅助技术读取）

## 六、反模式清单

以下做法**禁止出现**在小部件中：

| 反模式 | 原因 |
|--------|------|
| 显示任务数量徽标 | 制造焦虑（宪法第二条） |
| 添加/编辑/删除任何数据 | 增加认知成本（宪法第三条） |
| 内嵌滚动区域 | 违反铁律 1 |
| 使用红色/感叹号 | 制造焦虑（宪法第二条） |
| 显示完成百分比 | 暗示「你还差很多」 |
| 嵌入日历/时钟/天气 | 特性蔓延，小部件只做一件事 |
| 弹出通知/toast | 小部件是被动的，不主动打扰用户 |
