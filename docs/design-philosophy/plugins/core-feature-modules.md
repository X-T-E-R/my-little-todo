# 核心功能模块 — 设计原则

> 对应 `BUILT_IN_MODULES` 中主要作为 **主界面能力增强** 的一组模块（Kanban、时间胶囊、AI Coach、能量指示、Brain dump、理一理、高级筛选、时间感知等）的设计原则摘要。  
> **窗口上下文 / 桌面小部件 / MCP** 另有专文：[window-context.md](./window-context.md)、[desktop-widget-design.md](./desktop-widget-design.md)、[mcp-integration.md](./mcp-integration.md)。

---

## 通用原则

1. **可关闭**：每个模块必须可通过 `module:<id>:enabled` 关闭，关闭后主流程不得崩溃、不得留下无效入口（例如隐藏 Tab、隐藏按钮）。
2. **默认反映「打扰程度」**：越可能打断心流或增加噪音的，默认越偏关（如时间胶囊）；越属于主路径的，默认偏开（如 Kanban、AI Coach）。
3. **不制造焦虑**：与 [设计宪法](../00-design-constitution.md) 第二条一致——避免用未完成总数、红色警告、惩罚式 copy 作为默认激励。

## Kanban（`kanban`）

**默认**：开启  

**原则**：

- 看板是 **任务视图的一种**，不是第二套产品；列与卡片语义应复用核心任务模型（状态、列字段等），避免平行宇宙。
- 关闭模块时：**导航中移除「Tasks / 看板」入口**，且若当前正在该视图，应温和跳回「此刻」等安全视图。
- 交互上保持 **低配置启动**：用户不必先学列定义才能用；进阶配置可作为渐进披露。

---

## 时间胶囊（`time-capsule`）

**默认**：关闭  

**原则**：

- **强打扰能力 → 默认关**：随机以 Toast 等形式拉回旧内容，可能打断专注，必须由用户明确打开。
- **轻量、可忽略**：展示应是「温柔提醒」而非「你欠了多少」；频次与复制需克制，避免愧疚驱动。
- 与 AI Coach 的「欢迎回来」等区分：**时间胶囊偏「过去的火花」**，不是考勤或打卡。

---

## AI Coach（`ai-coach`）

**默认**：开启  

**原则**：

- **提议，不裁决**（宪法第四条）：Coach 相关文案与时机应是建议型，不替用户做不可逆决策。
- **低侵入**：例如长间隔回访时的轻量提示可以存在，但不应霸屏、不应默认声音打扰。
- **可关闭**：关闭后应停止依赖 Coach 时间线的体验路径，不应出现大量「功能残缺」的恐吓文案。

---

## 能量指示（`energy-indicator`）

**默认**：开启  

**原则**：

- 面向 **状态与节奏**，而非道德评判：不用「你太懒了」一类叙事；能量是中性信息。
- 与专注模式等结合时：**辅助用户选择难度与节奏**，不增加「又失败了一次」的压力。
- 视觉与动效应 **克制**，默认不抢占主内容区注意力。

---

## Brain dump（`brain-dump`）

**默认**：开启  

**原则**：

- **倒想法优先于分类**：入口与交互应极快（宪法第三条：添加一件事不超过约 3 秒的取向）。
- **不要求用户当场结构化**：后续整理可以在流 / 任务里完成，Brain dump 本身保持低摩擦。
- 关闭后：快捷入口隐藏，且不应在空状态用大面积文案指责用户未开启。

---

## 理一理（`think-session`）

**默认**：开启  

**原则**：

- **与 Brain dump 分工**：Brain dump 是秒级逐行写入 Stream；理一理是会话式 Markdown 长文，独立表 `think_sessions` 持久化。
- **不强迫计划**：三种起步（自由写 / 帮我看看 / 编排）仅作引导；用户可直接在编辑器里输入。
- **AI 可选**：未配置 API 时，本地编排与规则化 Discovery 仍可用；行动提取可降级为启发式。
- 关闭后：隐藏「流」内模式切换、此刻「我先理一理」入口与快捷键 `Ctrl+Shift+T`。

详见 [think-session.md](./think-session.md)。

---

## 时间感知（`time-awareness`）

**默认**：开启  

**原则**：

- **计划是建议不是合同**：固定时间块用于「上下文提示」与推荐加权，不替用户打卡或惩罚违约。
- **零配置也有价值**：即使未维护课表，也可从「此刻」接受/拒绝记录中学习时段偏好。
- **关闭后**：不显示时间表相关提示与推荐加权；看板日历不再叠加固定块（数据仍保留）。

---

## 高级筛选（`advanced-filter`）

**默认**：关闭  

**原则**：

- **进阶能力默认关**：列表页上的复杂筛选会增加决策与视觉噪音；只给需要「在大量任务中定位」的用户。
- 开启后：筛选控件应 **可发现但不过分占据首屏**；与「此刻」等简化视图哲学兼容——不要逼所有用户先学筛选器。
- 关闭时：流/列表行为保持简单默认筛选，不出现「半残」错误状态。
