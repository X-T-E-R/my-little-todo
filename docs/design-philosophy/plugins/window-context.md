# 窗口上下文（`window-context`）— 设计原则

> **模块 ID**：`window-context`  
> **典型环境**：Tauri **Windows** 桌面（前台窗口监听）；与 [桌面小部件](./desktop-widget-design.md) 共享部分系统能力时，以代码中的平台判断为准。

---

## 一、定位

1. **桥接「用户正在做什么」与「应用内角色/任务上下文」**  
   通过操作系统前台窗口信息（标题、进程等），匹配用户预先配置的规则，从而推断当前情境下更相关的 **角色或任务集合**。

2. **不是监控软件**  
   产品目的不是记录用户全天行为轨迹供第三方使用，而是 **在本地、为用户自己的任务服务**，且仅在用户启用模块时参与前台桥接。

3. **与桌面小部件的关系**  
   `desktop-widget` 与 `window-context` 可同时开启；二者都可能依赖前台信息时，应 **共享同一套前台桥接与存储**，避免重复监听与状态分叉（实现见 `useForegroundBridge` 等）。

---

## 二、设计原则

### 1. 默认保守

- **默认关闭**：涉及系统前台与隐私感知，默认 `defaultEnabled: false`，由用户显式开启。

### 2. 用户可控与可解释

- 匹配规则、角色映射等配置应 **可编辑、可删除**，并在 UI 中能用人话说明「在什么窗口下会切到哪类上下文」。
- 匹配失败时行为应 **可预期**（回退到用户手动选中的角色等），不静默切换到令人困惑的状态。

### 3. 最小必要数据

- 只采集匹配所需的最小字段（如标题模式、进程名）；不在产品层鼓励「全量上传窗口历史」。
- 与 [设计宪法](../00-design-constitution.md) 中的信任与认知成本原则一致：每多一层隐式推断，就要多一层可解释性。

### 4. 平台诚实

- 非 Tauri 或非支持平台：**不假装功能存在**；设置页说明限制，相关桥接不启动。

### 5. 与不制造焦虑一致

- 上下文条、角标等展示 **不应用红色倒计时或「你切换太频繁」式指责**；情境是提示，不是考勤。

---

## 三、与 `ContextBar` / 小部件

- 主窗口内的 **上下文展示条**应简洁：显示当前推断的角色/情境状态即可，避免堆叠调试信息。
- 小部件侧若显示情境，须遵守 [桌面小部件设计规范](./desktop-widget-design.md) 的铁律（信息密度、不滚动、不制造焦虑等）。

---

## 四、维护提示

- 前台 API 或权限变更时：更新本文件「平台诚实」小节与设置文案。
- 新增与窗口信息相关的设置键时：在本文档登记用途，避免散落魔法字符串。
